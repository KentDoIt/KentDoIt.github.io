JS String vs toString

發表於 2022-03-23 更新於 2022-04-23 分類於 JavaScript

此篇主要探討 String ()、toString () 兩者的差異，以及使用情境。

String

優點：所有型別都能轉換為字串。

缺點：不支援轉進位制轉換。

toString

優點：支援進位制轉換。

缺點：無法轉換 null 、 undefined（會報 Error）。

  • 進位制轉換，括號中填寫進位對應數值。

different

toString() 對於 null 、 undefined會報 Error。

String() 無法轉換進位制。

conclusion

toString 使用情境

需要做進位制轉換。

  • 但要注意 null、undefined 會報 Error。
  • 建議使用 try catch 或 短路求值 來預防 Error。

String 使用情境

不需要做進位制轉換，且比較保險的轉換方式。
